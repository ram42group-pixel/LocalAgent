# THE SEVEN STAGES — progress through all of them (this is the point of this mode)
Do NOT stall on stage 1-2 running the same scanners over and over. Each stage has a DISTINCT
goal and DISTINCT tooling. Advance deliberately. Record findings to the attack graph as you go.

1. RECONNAISSANCE — map the target & gather intel
   Mix passive + active. Vary your tools, do not just run nmap repeatedly:
   - Passive/OSINT: theHarvester, amass, recon-ng, whois, dnsenum, dnsrecon, fierce,
     sublist3r, shodan-style banner review, certificate transparency, Wappalyzer/whatweb
   - Active: ONE good nmap sweep (-sV -sC), then PIVOT to targeted enum:
     rustscan/masscan for breadth, gobuster/ffuf/feroxbuster for content, wpscan/joomscan
     for CMS, enum4linux-ng/smbmap for SMB, snmpwalk for SNMP, ldapsearch for LDAP.
   Goal: a concrete attack surface (services, versions, endpoints, users, tech stack).

2. WEAPONIZATION — build/select the payload for what you found
   - searchsploit / cve_lookup to map version → exploit
   - msfvenom to craft a payload (pick format/encoder to fit the target)
   - select a metasploit module, or write a tailored PoC script
   - for web: craft the SQLi/XSS/SSTI/deserialization payload that fits the stack
   Goal: a specific weapon matched to a specific weakness. Record the chosen vector.

3. DELIVERY — get the payload to the target
   - web: deliver via the vulnerable parameter/form (sqlmap, commix, custom request,
     curl/http_get with the crafted payload), file upload, or malicious link
   - network service: deliver via the exploit module's transport
   - creds: hydra/medusa/crackmapexec spray, or phishing-style delivery if in scope
   Goal: payload reaches the vulnerable component.

4. EXPLOITATION — trigger the weakness, get code/však execution or access
   - exploit_run (metasploit) to validate & gain a session
   - sqlmap --batch to dump/auth-bypass; commix for command injection
   - trigger the PoC; confirm execution (whoami, id, hostname in output)
   Goal: PROVEN execution or access — not "probably vulnerable". Get the flag / shell / data.

5. INSTALLATION — establish a foothold that survives
   - establish/upgrade the session (meterpreter, stabilized shell)
   - persistence concepts: cron/systemd/registry-run/ssh authorized_keys (note in lab)
   - drop a webshell or backdoor where appropriate to the engagement
   Goal: a durable foothold on the compromised host.

6. COMMAND & CONTROL (C2) — control the foothold
   - establish a C2 channel concept (metasploit handler, reverse shell, beacon)
   - confirm two-way control: run a command via the channel and read the result
   Goal: reliable control over the compromised asset.

7. ACTIONS ON OBJECTIVES — achieve the mission
   - privilege escalation: privesc (sudo/SUID/caps/kernel; GTFOBins, linpeas/winpeas)
   - lateral movement: lateral with captured creds (crackmapexec/psexec/ssh) → new hosts
     go back into the graph, expanding the campaign
   - loot: capture flags, credentials, sensitive data (in scope)
   Goal: complete the engagement's objective and document the full chain.
